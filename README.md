# AGRI-FORGE

> **Edge-native autonomous crop scouting for farmers.** A Jetson-hosted drone platform that flies itself, detects crop disease and stress in real time, maps microclimates across a field, and produces a 3D digital twin — all without a cloud connection.

AGRI-FORGE is the agricultural branch of the Firmament drone platform. Where the security branch hunts people and vehicles, AGRI-FORGE hunts **fungal disease, pest damage, nutrient stress, and weed pressure**, and turns every flight into a scouting report a farmer can act on before symptoms become visible.

The entire perception and decision stack runs **on-device** on a Jetson Orin Nano. No cloud, no subscription, no internet required in the field. The farmer connects to the drone's WiFi hotspot, taps a field zone on their phone, and gets an AI-graded health map in minutes.

## Architecture at a glance

```
DRONE (Jetson Orin Nano, runs entirely offline)
├── SIYI A8 4K camera ─→ RTSP ─→ image publisher
├── YOLOv8-nano ───────→ crop-class detection (11 ag classes)
├── Temporal reasoning ─→ persistence across flights
├── Cosmos Reason 2 ────→ agronomic assessment (VLM)
├── Ag perception node ─→ health scoring, zone correlation
├── BME280 sensor ──────→ humidity / temp / pressure
│                        → VPD, dew point, fungal risk
├── Zone mission planner → lawnmower grid waypoints
├── Mission logger ─────→ SQLite (~/firmament_data/missions.db)
└── Dashboard server ───→ FastAPI, port 8090, serves farmer PWA

FARMER (any phone/tablet, connects to drone hotspot)
└── Opens http://drone:8090 → map, detections, reports, verify UI

WORKSTATION (offline, USB data transfer)
├── Photogrammetry pipeline (OpenDroneMap)
│   └── orthomosaic, DSM, DTM, 3D mesh, point cloud
├── Training data merger (synthetic + farmer-verified)
└── YOLOv8 fine-tuning → deploy updated weights to Jetson
```

## What a farmer actually sees

1. Opens their phone, connects to the drone's WiFi.
2. Dashboard shows their fields and zones on a map of **their own aerial imagery** (generated by previous photogrammetry missions, stored offline on the Jetson).
3. Taps "NW Quarter" and hits **Scout**.
4. Drone takes off and flies an autonomous lawnmower grid.
5. Live during the flight:
   - YOLO flags crop anomalies as they appear on camera
   - BME280 maps humidity, temperature, and VPD across the zone
   - Ag perception node correlates detections with environmental conditions
   - Severe finds trigger the VLM for a written agronomic assessment
6. Drone lands, farmer reviews:
   - Zone health score (e.g., `NW Quarter: 72/100`)
   - Pinned detections on the map, each with AI reasoning
   - Environmental overlay: humidity hot spots, VPD hot spots, fungal risk zones
   - Scouting report: *"URGENT: fungal disease detected in rows 12–18, canopy humidity 89% RH, scout on foot immediately."*
7. Farmer confirms, rejects, or relabels each detection — this feeds the training flywheel.

## Key capabilities

### 🌾 Crop-class detection with temporal persistence

Eleven agricultural detection classes (disease, pest damage, nutrient stress, weeds) trained on a mix of synthetic data and farmer-verified real samples. The same area seen across multiple flights is tracked for **progression** — a spot that looks worse this week than last week is flagged as escalating, not just present.

### 💨 Microclimate sensing via BME280

Weather stations measure humidity at 6 feet above the ground. The drone measures it **at the crop canopy**, across the field, geo-tagged. Three readings combine into actionable intelligence:

- **Canopy humidity** — predicts where fungal disease will appear *before* it's visible
- **Vapor Pressure Deficit (VPD)** — the metric advanced growers actually use for irrigation decisions. Below 0.4 kPa → air saturated, fungal risk. Above 1.6 kPa → plants closing stomata, water stress. Spatial VPD maps at this price point don't exist elsewhere.
- **Pressure trends** — a 5+ hPa drop in one hour means incoming weather; drone auto-RTLs before it arrives.

### 🗺️ Photogrammetry → digital twin

RGB camera + dense flight pattern → OpenDroneMap pipeline produces:

- **Orthomosaic** — georeferenced 2D map of the field at ~2 cm/pixel
- **DSM / DTM** — surface and terrain elevation models (crop height, drainage analysis)
- **3D mesh** — textured 3D model the farmer can orbit in a browser
- **Point cloud** — LAS format for volumetric analysis
- **Offline tiles** — the orthomosaic becomes the dashboard's basemap, so the drone flies with imagery of **the farmer's own field**, not generic satellite.

No LiDAR required. No cloud processing required.

### 🔁 Farmer-verify training flywheel

- **Phase 1 — Bootstrap**: synthetic data factory generates 500+ training images for the 11 classes. Initial YOLOv8-nano trained on synthetic only.
- **Phase 2 — Field validation**: fly with the synthetic-trained model, dashboard presents each detection for farmer confirm/reject/relabel, verified annotations exported via USB.
- **Phase 3 — Merged training**: merge real annotations with synthetic at 3× real-weight, fine-tune on the blend, redeploy weights, repeat. Each farmer-flight cycle makes the model more accurate on *that farm*.

### 🛰️ Zero-interference retrofit

The scouting subsystem (`scout_dashboard/`) is designed to bolt onto an existing Firmament pipeline without touching it. The mission logger is a **passive ROS 2 subscriber** — adding it has zero effect on upstream publishers. The dashboard server runs on its own port. SQLite in WAL mode allows the logger to write while the dashboard reads. No topic renames, no launch-file edits, no risk to a working flight stack.
